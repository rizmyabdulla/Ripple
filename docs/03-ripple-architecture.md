# Ripple architecture proposal

## 1. Architecture goals

Ripple is a model family with a shared decoder and latent contract:

- **Ripple-VC Edge:** continuous streaming voice conversion.
- **Ripple-VC Quality:** same graph with one optional future frame and wider channels.
- **Ripple-TTS:** a later text/prosody frontend that drives the same latent contract and decoder.
- **Ripple-Server:** optional larger speaker/style and semantic planners, outside the edge v1 scope.

The edge architecture is selected for batch-one CPU latency, bounded state, quantization, and export. It does not include an autoregressive codec language model.

## 2. End-to-end data flow

```text
Target reference (once or infrequently)
  -> enrollment encoder
  -> identity vector + compact style-token bank
  -> cached speaker profile

Source PCM, 24 kHz, 20 ms chunks
  -> causal analysis encoder
  -> bounded semantic mixer
  -> semantic interface S[t]

Source PCM
  -> causal prosody estimator
  -> online speaker-normalized prosody P[t]

S[t] + P[t] + cached target profile
  -> causal source-filter decoder
  -> 20 ms output PCM
  -> optional watermark/post-limiter
```

## 3. Versioned latent contract: RIF-1

The Ripple Intermediate Features contract is the key to future TTS.

At 50 Hz:

- `semantic_soft[t]`: a probability distribution over 256–512 supervised semantic classes.
- `semantic_embed[t]`: expected class embedding projected to 128 dimensions.
- `prosody[t]`: normalized log-F0, voiced probability, periodicity/confidence, log-energy, and first-order deltas.
- `timing[t]`: implicit one-frame cadence in VC; later generated by the TTS duration model.

Per target profile:

- `speaker_global`: 192-dimensional identity/timbre vector.
- `speaker_tokens`: 4–8 vectors of 64 dimensions summarizing reference-local timbre and phonation.
- `pitch_profile`: robust target pitch range statistics, used only when target-range remapping is enabled.
- schema version, sample rate, normalization constants, and checksum.

Why soft semantic classes:

- They retain uncertainty and coarticulation better than hard units.
- They remain directly supervised by offline teachers.
- A later text model can predict the same distributions.
- Expected embeddings use standard matrix multiplication and avoid a fragile runtime codebook.
- Hard or FSQ auxiliary tokens can be added without changing the decoder input schema.

The decoder must be trained with semantic temperature/noise and occasional hard-token replacement so it remains robust when a future text frontend produces sharper distributions.

## 4. Source analysis encoder

### Proposed structure

- 24 kHz input, normalized float PCM.
- Initial causal Conv1d, kernel 7.
- Four causal residual stages with total stride 480, producing 50 Hz features.
- Stride plan selected by kernel benchmarking, for example `(3, 4, 5, 8)`.
- Depthwise causal convolutions plus pointwise expansion/projection.
- RMSNorm or channel-wise normalization that does not depend on future frames.
- SiLU/PReLU-class quantization-friendly activations.
- 128–192 latent channels.

### Why not copy SoundStream exactly

SoundStream proved the pattern, but Ripple should replace transposed-convolution assumptions, evaluate 24 kHz, use depthwise separable blocks, and expose every ring buffer in the state schema. The exact stride plan is a hardware-search variable, not a fixed research truth.

### Teacher supervision

The encoder predicts a shared semantic target distilled from a teacher ensemble:

- multilingual HuBERT/XLS-R for phonetic coverage;
- WavLM for robust speech context;
- Whisper encoder or ASR posteriors for cross-lingual intelligibility;
- optional supervised tokens from a CosyVoice-like tokenizer.

Only the selected distilled target is used at runtime. Teachers are never exported.

## 5. RippleMixer: bounded temporal context

The default mixer operates only at 50 Hz and uses four small blocks:

1. RMSNorm.
2. Causal depthwise Conv1d with a 7–15-frame kernel.
3. Gated pointwise feed-forward projection.
4. Every second block: causal local attention over the previous 16–32 frames.
5. Optional four-slot summary memory updated at a coarse cadence.

State is fixed:

- convolution rings;
- a fixed K/V ring for local attention;
- fixed summary-memory vectors.

No right context is used in edge mode. Quality mode permits one 20 ms future frame through an explicit mask and separate artifact.

### Why this hybrid

- Causal convolution handles local speech dynamics efficiently.
- Small fixed-window attention improves phonetic consistency without an unbounded cache.
- All operators have mature export/runtime support.
- At 50 Hz, a 32-frame attention window covers 640 ms while remaining cheap.
- The summary memory gives long conversations bounded context without preserving raw history.

### Alternatives

- **Pure convolution:** mandatory ablation and likely smallest tier.
- **Emformer block:** credible bounded-memory alternative if it wins on long-session quality.
- **Mamba/RWKV:** research-only branch requiring a fused step operator and measured edge benefit.
- **Full Conformer/global attention:** rejected for v1 because it adds cache and export complexity without clear batch-one benefit.

## 6. Causal prosody lane

The prosody estimator is a small causal convolutional network trained to match robust offline labels:

- log-F0;
- voiced/unvoiced probability;
- periodicity or pitch confidence;
- log-energy;
- optional spectral tilt/aperiodicity.

It uses 40–80 ms of past context and no future context in edge mode. Targets may come from RMVPE/CREPE-class models plus signal-based estimators. YIN remains a deterministic fallback and ablation.

Streaming normalization:

- robust Welford-style running mean/variance on voiced frames;
- confidence-gated updates;
- warm-start from population priors;
- bounded adaptation rate to prevent drift after silence;
- explicit reset/continue policy at VAD boundaries.

The decoder receives normalized source prosody. A target-range mode maps normalized pitch into the enrolled target’s robust range while preserving contour. The default mode preserves source contour and relative range.

Why explicit prosody remains:

- semantic units underrepresent pitch and energy;
- it gives measurable, controllable behavior;
- it allows independent leakage and preservation tests;
- an NSF-style decoder can use F0 directly.

## 7. Target-speaker enrollment

Enrollment runs once per target clip or loads a precomputed speaker profile.

### Encoder

- ECAPA/CAM++-inspired lightweight causal-or-offline convolutional encoder.
- Global attentive statistics pooling for identity.
- Small learned-query token pool for 4–8 local style/timbre tokens.
- Reference augmentation for channel, gain, and duration robustness.

### Conditioning

- `speaker_global` drives AdaRMS/FiLM in every decoder scale.
- `speaker_tokens` are attended only at 50 Hz or injected through a pooled per-frame modulation.
- Identity and style pathways have separate losses and dropout.

### Why more than one vector

A single vector is efficient but discards phonation and fine spectral traits. Full reference cross-attention is expensive. A tiny cached token bank captures more detail while keeping per-frame cost bounded.

For the smallest edge tier, the token bank can be pooled into the global vector. Both modes share the profile schema.

## 8. Causal source-filter waveform decoder

### Proposed structure

- Input projection from semantic and prosody features.
- Harmonic excitation generated from normalized/remapped F0.
- Learned noise excitation gated by voicing/aperiodicity.
- Four causal upsampling stages to 24 kHz.
- Nearest/linear resize plus causal depthwise convolution or subpixel shuffle; no default transposed convolution.
- Residual gated convolution blocks with speaker AdaRMS/FiLM.
- Direct waveform output, DC blocker, and conservative limiter.

### Why source-filter

- Explicit F0 improves periodic structure and reduces the burden on the decoder.
- Harmonic/noise separation is compatible with StreamVC’s later NSF observation.
- It makes pitch failures easier to diagnose.
- A deterministic oscillator is cheap and exportable.

### Why not BigVGAN, HiFi-GAN, or DAC unchanged

- BigVGAN is a quality teacher but alias-free periodic blocks and wide channels require edge profiling.
- HiFi-GAN is effective but common transposed upsampling can create chunk/checkerboard artifacts.
- DAC’s common high-fidelity setup is not causal.

Ripple should distill from high-quality offline vocoders where useful while retaining its own causal decoder.

## 9. Disentanglement design

No single bottleneck is trusted. Ripple uses:

- stopped decoder gradients into the content encoder during early training;
- semantic teacher supervision;
- source timbre/formant perturbation inspired by Seed-VC and speaker-anonymization work;
- adversarial source-speaker classification on semantic features;
- invariance loss between original and timbre-shifted content;
- explicit prosody outside the semantic channel;
- target-speaker dropout and mismatched source/target training;
- leakage probes that are evaluated, not merely optimized.

Gradient stopping can be relaxed only after leakage remains within gate and an ablation shows quality benefit.

## 10. Latency and model budget

Target, not measured:

- one 20 ms frame of algorithmic cadence;
- zero future lookahead;
- compute p95 below 20 ms and preferably below 10 ms;
- end-to-end p95 below 40 ms on the reference mobile CPU;
- optional quality mode below 60 ms with one future frame;
- bounded state below 8 MB excluding weights;
- INT8 artifact below 40 MB.

Compute budget should be allocated in this order:

1. waveform decoder;
2. source encoder;
3. 50 Hz mixer;
4. prosody estimator;
5. speaker enrollment, which is outside steady-state processing.

## 11. Architecture decision summary

Selected:

- 24 kHz causal convolutional I/O;
- 50 Hz soft semantic interface;
- fixed-window conv/attention mixer;
- learned causal normalized prosody;
- global plus token-level speaker profile;
- source-filter direct waveform decoder;
- standard exportable operators.

Deferred:

- semantic FSQ/RVQ;
- Mamba/RWKV;
- chunk-aware flow decoder;
- autoregressive depth decoder;
- dialogue/translation modeling.

Rejected for edge v1:

- online SSL teachers;
- unbounded attention;
- non-causal codec/vocoder;
- multi-step generative sampling;
- full speech-language-model stack.
